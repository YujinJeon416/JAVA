# Factory Method Pattern

- 팩토리 메서드 패턴에서 템플릿 메서드 패턴이 사용됨을 안다.
- 팩토리 메서드 패턴에서의 구조와 구현의 분리를 이해한다.

## 요구 사항
- 게임 아이템과 아이템 생성을 구현하시오
    - 아이템을 생성하기 전에 데이터베이스에서 아이템 정보를 요청
    - 아이템을 생성 후 아이템 복제 등의 불법을 방지하기 위해, 데이터베이스에 아이템 생성 log를 남긴다.
- 아이템을 생성하는 주체를 framework.ItemCreator
- 아이템은 framework.Item 이라는 interface로 다뤄지도록 한다.
    - Item은 use 함수를 기본 함수로 가진다.
- 현재 아이템의 종류는 HP, MP 회복 물약

## 원리
- 서브 클래스에서 구체적인 오브젝트 생성 방법 결정
- 슈퍼클래스에서는 `abstract`하게 팩토리메서드(템플릿메서드)를 생성해둔다.
    - 이후 서브클래스를 통하여 추상화된 객체를 실제로 구현한다.
    - **Main클래스(클라이언트)에서는 `ItemCreator creator = new HpCreator();`**
        - 즉 `추상클래스 = 서브클래스`해주어, 추상클래스에 의존하여 서비스를 수행한다.
